# Notion 견적서 공유 서비스 (Invoce) MVP PRD

## 핵심 정보

**목적**: 프리랜서·소규모 사업자가 Notion에 작성한 견적서를 클라이언트에게 웹 링크 하나로 전달하고, 클라이언트는 별도 계정 없이 열람과 PDF 다운로드를 완결할 수 있게 한다.
**사용자**: Notion을 업무 도구로 쓰는 프리랜서·디자이너·개발자·소규모 에이전시 운영자, 그리고 링크를 전달받는 그들의 클라이언트(고객).

---

## 사용자 여정

```
1. [로그인 페이지] — 사업자(공급자)
   ↓ 이메일·비밀번호 입력 후 로그인

   신규 사용자 → [회원가입 페이지]
                 ↓ 가입 완료
               → [Notion 연동 페이지]
                 ↓ 토큰·DB ID 저장 완료
               → [내 견적서 목록 페이지]

   기존 사용자 → [내 견적서 목록 페이지]

2. [내 견적서 목록 페이지] — 사업자
   ↓ 견적서 목록 확인
   ↓ "링크 복사" 버튼 클릭 → 클립보드에 공유 URL 복사
   ↓ 클라이언트에게 링크 전달 (이메일·메신저 등 외부 채널)

3. [견적서 공개 보기 페이지] — 클라이언트 (비로그인)
   ↓ 공유 링크 클릭 → 견적서 즉시 표시
   ↓ "PDF 다운로드" 버튼 클릭

4. [PDF 다운로드 완료] → 클라이언트 작업 종료

[보호 페이지 접근 시]
   비로그인 사용자가 내 견적서 목록·Notion 연동 페이지 접근
   → 자동 리디렉션 → [로그인 페이지]

[잘못된 공유 링크 접근 시]
   존재하지 않는 slug → 404 안내 표시
```

### Notion DB 쓰기 구조 (F012 견적서 작성)

별도 자체 DB 없이 Notion을 단일 데이터 소스로 유지하므로, 견적서 작성 시에도 사용자 계정용 DB 없이 Notion API로 직접 페이지를 생성한다.

| Notion DB | 역할 | 쓰기 시점 |
|-----------|------|----------|
| Invoices DB | 견적서 1건 = 페이지 1개 (제목·클라이언트명·발행일·유효기간·상태·공급자 정보·총 금액) | 항목 페이지 생성 완료 후, 마지막에 1회 생성 |
| Items DB | 견적 항목 1개 = 페이지 1개 (품목명·수량·단가), `Invoices` relation 속성으로 견적서 페이지와 연결 | 폼의 항목 개수만큼 먼저 생성 |

**쓰기 순서**: 항목 입력값 → Items DB에 각각 페이지 생성 → 생성된 항목 페이지 ID 목록을 Invoices DB 신규 페이지의 `항목` relation 속성에 연결 → 완료.

# 노션 기반 견적서 관리 시스템 MVP PRD

## 🎯 핵심 정보

**목적**: 노션을 데이터베이스로 활용하여 견적서를 관리하고, 클라이언트가 웹에서 조회 및 PDF 다운로드할 수 있는 시스템
**사용자**: 견적서를 발행하는 프리랜서/소규모 기업과 견적서를 받는 클라이언트

## 🚶 사용자 여정

### 견적서 작성자 (관리자)

```
1. 노션 데이터베이스
   ↓ 견적서 정보 입력

2. 노션에서 견적서 작성 완료
   ↓ 자동으로 고유 URL 생성

3. 클라이언트에게 고유 링크 전달
   ↓ 이메일/메신저로 공유

4. 완료
```

### 클라이언트 (견적서 수신자)

```
1. 이메일/메신저에서 링크 클릭
   ↓ 고유 견적서 URL 접속

2. 견적서 조회 페이지
   ↓ 견적서 내용 확인

3. PDF 다운로드 버튼 클릭
   ↓ PDF 생성 및 다운로드

4. 완료 → 견적서 파일 저장/인쇄 가능
```

## 📦 핵심 구현 로직

### 1. Notion 데이터 조회 (Server Component)

```typescript
// app/invoice/[id]/page.tsx
import { Client } from '@notionhq/client'

const notion = new Client({ auth: process.env.NOTION_API_KEY })

async function getInvoice(pageId: string) {
  const page = await notion.pages.retrieve({ page_id: pageId })
  const blocks = await notion.blocks.children.list({ block_id: pageId })
  return { page, blocks }
}
```

### 2. PDF 생성 (API Route)

```typescript
// app/api/generate-pdf/route.ts
import { PDFDocument } from '@react-pdf/renderer';

export async function POST(req: Request) {
  const { invoiceData } = await req.json();
  const pdfDoc = <InvoicePDF data={invoiceData} />;
  const blob = await pdf(pdfDoc).toBlob();
  return new Response(blob, {
    headers: { 'Content-Type': 'application/pdf' }
  });
}
```
